# 西安游迹（XI'AN TRAVEL GUIDE）

一款以西安示意地图为入口，按区域查看景点、美食，并辅助个人旅行规划的轻量微信小程序。

项目采用微信原生小程序与原生 `map` 组件开发，所有旅行内容均由本地 JavaScript 数据驱动，不依赖服务器、数据库、账号系统或第三方地图 SDK。

## 已实现功能

### 地图探索

- 首页西安地图与区域入口
- 横屏全屏地图
- 5 个精选景点 marker
- 用户主动定位与当前位置显示
- `map-points.js` 地图特殊地点系统
- 景点详情页周边探索地图
- 微信原生地图位置与导航能力

当前精选景点：

- 西安钟楼
- 大雁塔·大慈恩寺
- 陕西历史博物馆
- 洒金桥
- 大唐不夜城

当前特殊地点为西安奥体中心。特殊地点只在首页和全屏地图中显示，不进入景点详情、区域攻略或周边探索。

### 景点与美食

- 5 个旅行区域及区域攻略
- 区域页景点、美食手风琴分类，默认全部收起并互斥展开
- 13 个景点详情
- 开放时间、门票、建议游览时长和注意事项
- 9 条美食内容及推荐店铺信息
- 景点参考位置和 3 公里内周边景点展示
- 具有可靠坐标的美食参考位置
- 景点、美食详情页使用 `wx.openLocation` 打开微信地图

### 实时搜索

- 首页地图下方提供内嵌实时搜索，不跳转搜索页面
- 区域页提供限定当前区域的内嵌实时搜索
- 景点与美食统一按名称进行包含匹配，名称开头匹配优先
- 半透明联想面板最多展示 5 条结果
- 景点结果进入景点详情，美食结果进入美食详情
- `search-utils.js` 为首页、区域页和保留的独立搜索页提供统一搜索结果结构

## 核心页面流

```text
首页 home
├── 区域卡片 → region?id=区域ID
├── 实时搜索景点与美食 → detail / food-detail
├── 精选景点 marker → detail?id=景点ID
├── 特殊地点 marker → 仅地图展示
├── 全屏按钮 → full-map
└── 定位按钮 → 获取并显示用户当前位置

区域攻略 region
├── 当前区域实时搜索 → detail / food-detail
├── 景点分类 → 点击后展开或收起景点列表
└── 美食分类 → 点击后展开或收起美食列表

景点详情 detail
├── 查看参考位置 → place-map?id=景点ID
└── 打开地图导航 → wx.openLocation

景点地图 place-map
└── 附近景点 marker → detail?id=附近景点ID

美食详情 food-detail
├── 查看参考位置 → food-map?id=美食ID
└── 打开地图导航 → wx.openLocation
```

## 页面结构

```text
miniprogram/pages/
├── home/          首页地图、实时搜索、区域入口和用户定位
├── full-map/      横屏全屏地图
├── region/        区域内搜索及景点、美食折叠列表
├── search/        保留的独立实时搜索页，当前无首页或区域入口
├── detail/        景点详情
├── place-map/     当前景点与附近景点地图
├── food/          美食分类展示
├── food-detail/   美食详情
├── food-map/      美食参考位置地图
└── index/         初始工程保留页面，当前不是启动页
```

## 数据与地图配置

```text
miniprogram/
├── data/
│   ├── regions.js       旅行区域
│   ├── places.js        景点内容与坐标
│   ├── foods.js         美食内容与可选坐标
│   ├── map-config.js    地图中心、缩放、精选地点和周边参数
│   └── map-points.js    只在地图展示的特殊地点
└── utils/
    ├── map-utils.js       坐标校验、距离计算和附近景点筛选
    ├── location-utils.js  微信原生地图打开能力
    └── search-utils.js    景点、美食名称匹配和统一联想结果
```

当前内容规模：

- 5 个旅行区域
- 13 个景点，全部具有 GCJ-02 参考坐标
- 9 条美食，其中 2 条具有已确认的参考坐标
- 1 个地图特殊地点

## 本地运行

1. 克隆仓库。
2. 打开微信开发者工具，导入仓库中的 `miniprogram/` 目录。
3. 使用可用的小程序 AppID 或测试号完成项目配置。
4. 编译后从首页检查地图、区域卡片和页面跳转。
5. 定位功能只会在用户主动点击时申请权限；请分别测试允许、拒绝和系统定位关闭的情况。
6. 横屏全屏地图、原生地图层级和 `wx.openLocation` 建议在真机中回归。

`miniprogram/project.config.json` 可能包含本机 AppID 和开发者工具配置，按项目约定不纳入功能提交。

## 产品边界

- 地图坐标仅用于旅行参考，实际入口和门店信息以官方或商家最新信息为准。
- 小程序不自行实现路线规划，也不接入第三方地图 SDK。
- 不保存用户位置，不进行后台持续定位。
- 不使用服务器、数据库、登录、收藏或评论系统。
- 美食已使用本地 JPG；景点图片仍使用占位状态。
