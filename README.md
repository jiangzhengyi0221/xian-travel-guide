# 西安游迹（XI'AN TRAVEL GUIDE）

一款以西安地图为入口，按区域查看景点与美食，并辅助个人旅行规划的轻量微信小程序。

项目采用微信原生小程序和原生 `map` 组件开发，旅行内容由本地 JavaScript 数据驱动，不依赖服务器、数据库、账号系统或第三方地图 SDK。

## 当前版本

- 版本：**V0.1.2.1**
- 区域：**6 个**
- 景点：**21 个**
- 美食：**9 条**
- 地图特殊地点：**1 个**

## 已实现功能

### 首页地图与搜索

- 使用微信原生 `map` 展示西安地图。
- 展示 5 个精选景点 marker 和西安奥体中心特殊地点。
- 支持用户主动定位和当前位置显示。
- 支持进入横屏全屏地图。
- 搜索框位于首页地图卡片顶部。
- 首页搜索范围仅包含 `places.js` 中的景点。
- 景点名称按前缀优先、包含匹配生成最多 5 条联想结果。
- 点击搜索结果后不跳转详情页，而是将地图移动到景点坐标。
- 定位成功后生成一个临时搜索 marker，新结果会替换旧结果。
- 点击精选景点或临时搜索 marker 可进入景点详情。

### 区域与内容

- 6 个旅行区域：雁塔区、碑林区、莲湖区、钟鼓楼区域、临潼区、未央区。
- 区域页按 `regionId` 展示景点与美食。
- 景点、美食分类默认收起并互斥展开。
- 区域页搜索当前区域内的景点与美食，点击结果进入对应详情页。
- 保留独立搜索页，继续复用公共搜索工具，但首页和区域页没有常规入口跳转至该页。

### 景点

- 21 个景点详情，均具有 GCJ-02 参考坐标。
- 展示简介、详细介绍、地址、开放时间、门票、建议游览时长、推荐理由、标签和注意事项。
- 统一使用 `reservation` 对象维护预约状态、官方渠道、操作说明、来源和核实日期。
- 新增景点使用 `locationMeta` 记录坐标系、坐标来源、定位说明和核实日期。
- 景点详情支持查看参考位置、3 公里内周边景点和微信原生地图导航。
- 详细介绍默认显示约四行摘要，可点击“查看详细介绍”展开，再次点击收起。

### 美食

- 9 条美食内容及推荐店铺信息。
- 使用本地 JPG 封面和详情轮播图片，支持全屏预览。
- 具有可靠坐标的美食可查看参考位置并打开微信地图导航。

## 核心页面流

```text
首页 home
├── 景点搜索 → 地图定位 + 临时 marker
├── 精选景点 / 临时 marker → detail?id=景点ID
├── 区域卡片 → region?id=区域ID
├── 特殊地点 marker → 仅地图展示
├── 全屏按钮 → full-map
└── 定位按钮 → 显示用户当前位置

区域攻略 region
├── 当前区域景点搜索 → detail?id=景点ID
├── 当前区域美食搜索 → food-detail?id=美食ID
├── 景点分类 → 展开 / 收起景点列表
└── 美食分类 → 展开 / 收起美食列表

景点详情 detail
├── 详细介绍 → 四行摘要 / 展开 / 收起
├── 预约信息 → 官方渠道说明
├── 查看参考位置 → place-map?id=景点ID
└── 打开地图导航 → wx.openLocation

景点地图 place-map
└── 附近景点 marker → detail?id=附近景点ID

美食详情 food-detail
├── 查看参考位置 → food-map?id=美食ID
└── 打开地图导航 → wx.openLocation
```

## 页面结构

```text
miniprogram/pages/
├── home/          首页地图、景点搜索定位、区域入口和用户定位
├── full-map/      横屏全屏地图
├── region/        区域内景点、美食搜索及折叠列表
├── search/        保留的独立实时搜索页
├── detail/        景点详情与介绍折叠
├── place-map/     当前景点与附近景点地图
├── food/          美食分类展示
├── food-detail/   美食详情
├── food-map/      美食参考位置地图
└── index/         初始工程保留页面，当前不是启动页
```

## 数据与工具

```text
miniprogram/
├── data/
│   ├── regions.js       6 个旅行区域
│   ├── places.js        21 个景点、坐标和预约信息
│   ├── foods.js         9 条美食与可选坐标
│   ├── map-config.js    地图中心、缩放、精选地点和周边参数
│   └── map-points.js    仅用于地图展示的特殊地点
└── utils/
    ├── map-utils.js       坐标校验、距离计算和附近景点筛选
    ├── location-utils.js  微信原生地图打开能力
    └── search-utils.js    景点、美食名称匹配和统一联想结构
```

## 本地运行

1. 克隆仓库。
2. 使用微信开发者工具导入仓库中的 `miniprogram/` 目录。
3. 使用可用的小程序 AppID 或测试号完成项目配置。
4. 编译后检查首页地图、搜索定位、区域卡片和详情页。
5. 定位功能只会在用户主动点击时申请权限，请分别测试允许、拒绝和系统定位关闭。
6. 横屏地图、原生地图层级、marker、`wx.openLocation` 和长文本折叠建议在真机回归。

`project.private.config.json`、微信缓存和本机临时文件不属于功能提交。

## 产品边界

- 地图坐标为 GCJ-02 旅行参考位置，实际入口以官方最新信息为准。
- 小程序不自行实现路线规划，也不接入第三方地图 SDK。
- 不保存用户位置，不进行后台持续定位。
- 不使用服务器、数据库、登录、收藏或评论系统。
- 开放时间、门票、预约和门店信息可能变化，临近出行应再次核验。
- 美食已使用本地 JPG；景点图片仍处于占位状态。
