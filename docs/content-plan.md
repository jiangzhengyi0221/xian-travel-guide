## 3. 数据结构与关联规则

### 3.1 区域数据

`regions.js` 使用稳定英文 `id` 管理区域名称、简介和标签。景点与美食通过 `regionId` 关联区域；“钟鼓楼区域”是旅行浏览分区，不代表正式行政区划。

### 3.2 景点数据

`places.js` 是景点详情、精选 marker、周边探索和预约信息的唯一景点内容来源。当前 13 个景点均具有 GCJ-02 参考坐标和统一 `reservation` 对象，页面通过景点 `id` 查询详情，地图 marker 只保存必要展示数据并建立 marker ID 到景点 ID 的映射。

### 3.3 景点预约数据

预约对象统一保存：

- `status`：是否需要预约、是否有官方渠道或是否暂未确认。
- `type`：官方微信公众号、官方网页、无需预约或暂未确认。
- `platform`、`accountName`、`url`：已可靠确认的官方平台信息。
- `appId`、`path`：只有同时确认有效时才能用于微信小程序跳转；当前全部为空。
- `instructions`：面向用户的操作说明和限制提示。
- `sourceName`、`sourceUrl`、`verifiedAt`：官方依据和核实日期。

### 3.4 美食数据

`foods.js` 管理美食详情、推荐店铺和图片信息。经纬度是可选字段：只有坐标有效时才显示美食地图与导航入口。当前 9 条美食中，2 条具有已确认的参考坐标。`image` 保存列表封面路径，`images` 保存详情轮播路径；图片目录名与美食 ID 一一对应。

### 3.5 地图配置与特殊地点

- `map-config.js` 统一管理默认地图中心、默认缩放级别、定位后缩放级别、精选景点 ID、周边半径和数量上限。
- `map-points.js` 管理不属于景点内容体系、只需出现在地图上的特殊地点。
- 特殊地点不写入 `places.js`，不关联区域、不进入景点详情，也不参与 place-map 周边探索。

## 4. 地图展示逻辑

### 4.1 首页与全屏地图

首页和横屏全屏地图复用 `map-config.js` 中的 5 个精选景点：

1. `xian-bell-tower`：西安钟楼
2. `dayanta`：大雁塔·大慈恩寺
3. `shaanxi-history-museum`：陕西历史博物馆
4. `sajinqiao`：洒金桥
5. `datang-everbright-city`：大唐不夜城

marker 名称与坐标继续从 `places.js` 按 ID 获取。精选景点 marker 点击进入景点详情。特殊地点 marker 来自 `map-points.js`，当前西安奥体中心在首页和全屏地图显示，点击不跳转。

两个页面默认以西安市中心为地图中心，默认缩放级别为 12；用户主动定位成功后使用缩放级别 14。定位信息不保存，也不在页面加载时自动获取。

### 4.2 景点周边探索

- place-map 显示当前景点及附近景点。
- 从 `places.js` 中排除当前景点，筛选直线距离 3 公里内的地点，按距离升序取最多 5 个。
- 当前景点 marker 点击不跳转，附近景点 marker 点击进入对应详情。
- 地图使用 `include-points` 包含当前点与附近点并自动调整范围。
- 特殊地点与美食不参与周边景点筛选。

### 4.3 美食地图与导航

- 只有具有有效坐标的美食才显示地图和导航入口。
- food-map 展示当前美食参考门店 marker，点击后校验来源并返回对应 food-detail。
- 景点和美食详情通过微信原生 `wx.openLocation` 打开地图；景点使用景点名称，美食优先使用推荐店铺名。
- 无有效坐标时隐藏相关入口，不使用猜测位置。

## 5. 内容维护原则

### 5.1 景点内容

景点内容应保持以下信息完整：

- 名称和所属区域。
- 摘要与详细介绍。
- 地址和参考坐标。
- 开放时间、门票和建议游览时长。
- 推荐理由、标签和注意事项。
- 图片状态。
- 预约状态、官方渠道和操作说明。
- 官方来源、来源网址和核实日期。

开放时间、门票和预约规则可能变化，临近旅行日期时需要再次核对官方信息。

预约信息维护要求：

- 只采用景区官方网站、官方微信公众号、官方微信小程序或政府官方页面。
- 不使用第三方旅游平台作为官方入口。
- 不猜测小程序 `appId`、页面 `path`、网址或公众号名称。
- 官方网页无法满足小程序 `web-view` 业务域名要求时，只提供网址复制和明确说明。
- 每次重新核实后同步更新 `verifiedAt`。

### 5.2 美食内容

美食内容应保持以下信息完整：

- 名称、类型和所属区域。
- 摘要与详细介绍。
- 推荐店铺、地址和参考价格。
- 推荐理由、标签和注意事项。
- 图片状态。
- 可选的可靠参考坐标。

美食可能存在多家分店或门店变化。只有能够确认参考门店时才补充坐标，实际信息以商家最新情况为准。

### 5.3 图片内容

- 9 条美食已接入 `miniprogram/images/foods/` 下的 19 张本地 JPG。
- 区域页和美食列表使用封面图；美食详情使用带指示点的轮播，并支持点击全屏预览。
- 当前所有美食 JPG 路径均与真实文件及大小写一致，图片非空且可正常解码。
- 景点仍使用图片占位状态；后续接入前需要确认来源和使用权限，并进行压缩以控制小程序包体积。
- 图片风格应与当前米白、白色卡片和红褐色标题的视觉体系协调。
