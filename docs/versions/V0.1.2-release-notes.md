# 《西安游迹》V0.1.2 版本记录

> 项目名称：西安游迹（XI'AN TRAVEL GUIDE）
> 项目类型：微信原生小程序
> 版本号：V0.1.2
> 更新日期：2026-07-24
> 核心更新：扩充景点数据，补全地图坐标、开放时间、门票和预约信息

## 1. 版本背景

V0.1.1 完成首页景点搜索和地图定位后，项目需要增加更多西安景点，并保证新增内容能够直接复用现有区域、搜索、详情、参考地图和导航能力。

V0.1.2 以数据扩充和信息完善为主，不改变页面、公共搜索工具和地图工具的既有接口。

## 4. 坐标接入

8 个新增景点均补充：

- `address`
- `latitude`
- `longitude`
- `locationMeta`

坐标体系为 GCJ-02，可直接用于微信原生 `map`、首页搜索定位、景点参考地图和 `wx.openLocation`。

`locationMeta` 结构：

```js
{
  coordinateSystem: 'GCJ-02',
  coordinateSource: '',
  locationNote: '',
  verifiedAt: '2026-07-24'
}
```

字段用途：

- `coordinateSystem`：明确坐标系。
- `coordinateSource`：记录坐标查询与交叉核验来源。
- `locationNote`：说明选择游客入口、游客中心或代表性到达点的原因。
- `verifiedAt`：记录本次坐标核验日期。

`locationMeta` 仅用于内部维护，不改变现有页面与工具接口。

## 5. 开放时间与门票

8 个新增景点完善了：

- `openTime`
- `ticket`

数据表达延续现有项目风格：

- 对明确的当前安排记录具体信息。
- 对可能发生季节、节假日或活动调整的内容标注“通常”或“以官方公告为准”。
- 区分免费公共区域、收费区域和独立收费项目。
- 不使用“待补充”等过时占位文本。

开放时间和门票可能变化，临近出行仍需核对景区或场馆最新公告。

## 6. 预约信息

8 个新增景点均补充与现有项目兼容的 `reservation` 对象，包括：

- 预约状态。
- 渠道类型。
- 状态文案与标题。
- 平台或公众号名称。
- 操作说明。
- 信息来源和来源网址。
- 核实日期 `2026-07-24`。

数据原则：

- 不虚构不存在的官方小程序入口。
- 未可靠确认的 `appId`、`path` 或网址保持为空。
- 通常无需预约的景点明确说明高峰期、活动或独立项目可能采用其他规则。
- 未确认稳定官方线上入口的景点明确建议现场核对。

## 8. 功能影响

### 搜索

- 新增景点自动进入 `places.js` 搜索数据源。
- 首页可搜索新增景点并根据坐标定位地图。
- 区域页可在对应区域搜索新增景点并进入详情。
- 无需修改 `search-utils.js`。

### 地图

- 8 个新增景点均具有有效 GCJ-02 坐标。
- 可用于临时搜索 marker、景点参考地图、周边景点筛选和微信地图导航。
- 不改变首页默认精选 marker 配置。

### 详情页

- 新增景点复用现有详情页。
- 地址、开放时间、门票、预约、推荐理由、注意事项和标签可按现有结构展示。
