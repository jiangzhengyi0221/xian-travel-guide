# 《西安游迹》V0.1.1 版本总结

> 项目名称：西安游迹（XI'AN TRAVEL GUIDE）
> 项目类型：微信原生小程序
> 当前版本：V0.1.1
> 版本状态：已合并至 `main`
> 核心更新：完成首页地图搜索与景点定位交互

## 1. V0.1.1 新增功能

### 1.1 首页地图搜索

首页新增与地图视觉融合的景点搜索区域，搜索框位于地图卡片顶部。

支持：

- 点击搜索入口展开输入框。
- 输入景点名称时实时显示联想结果。
- 名称开头匹配优先。
- 最多展示 5 条搜索结果。
- 空关键词时隐藏结果面板。
- 无匹配内容时显示明确提示。
- 支持取消搜索并清空输入状态。

当前首页地图搜索范围仅包含 `places.js` 中的景点，不搜索区域和美食。

### 1.2 搜索结果地图定位

点击搜索结果后：

- 不进入景点详情页。
- 从 `places.js` 按景点 ID 查找完整数据。
- 校验景点经纬度是否合法。
- 将首页地图中心移动到目标景点。
- 将地图缩放级别调整为适合查看单个景点的级别。
- 清空并收起搜索状态。
- 坐标无效时显示友好提示。

### 1.3 临时搜索 Marker

搜索定位成功后，地图会增加一个临时 marker：

- marker 显示目标景点名称。
- 使用独立 marker ID，避免与精选景点和特殊地点冲突。
- 新的搜索结果会替换上一次临时 marker，不会不断累积。
- 如果目标已经是精选景点，会先隐藏同位置的原 marker，避免两个 marker 重叠。
- 点击临时 marker 后仍可进入对应景点详情页。

### 1.4 原有功能兼容

本次更新保留了：

- 首页精选景点 marker。
- 地图特殊地点。
- 用户主动定位。
- 横屏全屏地图。
- marker 点击进入景点详情。
- 区域卡片及区域攻略入口。
- 区域页和独立搜索页原有的景点、美食搜索。

公共搜索工具接口没有改变，因此未影响其他页面。

## 3. 技术实现

### 3.1 搜索数据流

```text
用户输入景点名称
        ↓
home.js 获取关键词
        ↓
buildSearchSuggestions({
  places,
  keyword,
  limit: 5
})
        ↓
返回统一景点联想结果
        ↓
用户点击“定位”
        ↓
根据结果 ID 查询 places.js
        ↓
校验 latitude / longitude
        ↓
更新地图中心、缩放级别和 markers
```

### 3.2 地图移动

首页地图继续使用数据绑定控制视野：

```js
{
  mapLatitude,
  mapLongitude,
  mapScale
}
```

搜索成功后，通过 `setData()` 更新：

- `mapLatitude`
- `mapLongitude`
- `mapScale`

该方式与现有用户定位功能保持一致，没有引入新的地图 SDK。

### 3.3 Marker 结构

首页 marker 分为三类：

```text
基础 markers
├── 精选景点 markers
└── 地图特殊地点 markers

临时 marker
└── 当前搜索结果 marker
```

关键常量：

```js
const SPECIAL_MARKER_ID_START = 1000
const SEARCH_RESULT_MARKER_ID = 2000
const SEARCH_RESULT_SCALE = 15
```

通过独立 ID 区间避免 marker 冲突。

搜索结果 marker 仍会写入 `markerIdToPlaceId` 映射，因此用户单独点击该 marker 时可以进入景点详情。

### 3.4 搜索工具兼容

`search-utils.js` 没有修改。

首页只传入景点数组：

```js
buildSearchSuggestions({
  places,
  keyword,
  limit: 5
})
```

区域页和独立搜索页仍可继续传入：

```js
buildSearchSuggestions({
  places,
  foods,
  keyword,
  limit
})
```

因此公共工具的参数、返回结构和原有调用方式均保持兼容。

### 3.5 数据原则

地图搜索遵循以下原则：

- 景点坐标来自本地 `places.js`。
- 使用 GCJ-02 参考坐标。
- 不调用服务器或云数据库。
- 不调用第三方地图 SDK。
- 不猜测区域或美食位置。
- 无有效坐标时不执行地图定位。
- 地图位置仅供旅行参考。
