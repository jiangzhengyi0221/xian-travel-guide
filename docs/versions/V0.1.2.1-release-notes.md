# 《西安游迹》V0.1.2.1 版本记录

> 项目名称：西安游迹（XI'AN TRAVEL GUIDE）
> 项目类型：微信原生小程序
> 版本号：V0.1.2.1
> 更新类型：景点详情页展示优化
> 核心更新：详细介绍由默认完整展开调整为“四行摘要 + 手动展开”模式

## 1. 优化背景

V0.1.2 完成新增景点、坐标及开放预约信息接入后，景点详情页中的 `description` 内容更加完整，但长篇介绍默认全部展开，导致页面高度明显增加。

用户进入详情页后，需要经过较长距离的滑动才能看到：

- 地址信息。
- 景点参考位置。
- 地图导航入口。
- 推荐理由。
- 地点标签和实用提醒。

V0.1.2.1 将“详细介绍”调整为可折叠展示，在保留完整景点数据的前提下缩短默认页面长度，让地图和实用信息更快进入用户视野。

## 2. 修改文件

本次优化仅修改景点详情页的三个文件：

```text
miniprogram/pages/detail/
├── detail.js
├── detail.wxml
└── detail.wxss
```

### `detail.js`

- 新增介绍展开状态 `isDescriptionExpanded`。
- 默认值为 `false`。
- 页面加载时明确恢复为折叠状态。
- 新增 `toggleDescription()`，负责在展开和折叠状态之间切换。

### `detail.wxml`

- 保留“详细介绍”标题和原始 `place.description` 数据绑定。
- 根据 `isDescriptionExpanded` 动态切换正文样式类。
- 增加“查看详细介绍 / 收起介绍”切换按钮。
- 仅在 `description` 存在时显示切换按钮。

### `detail.wxss`

- 增加四行摘要截断样式。
- 增加完整展开样式。
- 增加轻量级文字按钮样式。
- 保留原有字号、行高、两端对齐和段落换行效果。

本次未修改：

- `miniprogram/data/places.js`
- `miniprogram/data/regions.js`
- `miniprogram/utils/`
- 其他页面文件

## 3. 技术实现

### 3.1 页面状态

详情页新增布尔状态：

```js
isDescriptionExpanded: false
```

状态含义：

- `false`：显示四行摘要。
- `true`：显示完整详细介绍。

切换操作只更新当前页面实例，不写入全局状态、缓存或景点数据。

### 3.2 状态切换

用户点击介绍区域下方按钮时，`toggleDescription()` 对当前状态取反：

```text
false（四行摘要）
        ↓ 点击“查看详细介绍”
true（完整介绍）
        ↓ 点击“收起介绍”
false（四行摘要）
```

### 3.3 摘要实现

折叠状态通过 WXSS 多行截断实现：

```css
display: -webkit-box;
overflow: hidden;
-webkit-box-orient: vertical;
-webkit-line-clamp: 4;
```

实现过程中没有在 JavaScript 中截取字符串，也没有生成新的摘要数据。WXML 始终直接绑定完整的：

```text
place.description
```

展开状态恢复普通块级显示，并保留 `white-space: pre-wrap`，因此原始介绍中的段落和换行仍能正常呈现。

### 3.4 兼容设计

折叠状态只作用于“详细介绍”正文，不参与其他页面逻辑。

以下功能保持原样：

- 景点名称与简介。
- 所属区域。
- 开放时间、门票和建议游览时长。
- 预约状态及预约按钮。
- 地址显示。
- 景点参考位置。
- 微信地图导航。
- 推荐理由。
- 地点标签。
- 实用提醒。

## 4. 用户体验变化

### 优化前

- 详细介绍进入页面后立即完整展开。
- 长内容占据大量首屏之后的页面空间。
- 地址、地图和推荐理由需要较长距离滑动才能看到。

### 优化后

- 默认只显示约四行介绍摘要。
- 用户可以快速浏览景点核心信息和后续实用内容。
- 点击“查看详细介绍”后可阅读完整内容。
- 展开后按钮自动变为“收起介绍”。
- 再次点击即可恢复紧凑布局。
- 每次重新进入详情页时默认保持折叠状态。

## 6. 后续计划

### 6.1 完成微信端视觉验收

在微信开发者工具和真机上覆盖长介绍、多段落介绍及不同屏幕尺寸，确认四行截断和展开交互稳定。

### 6.2 评估短介绍场景

当前所有非空 `description` 都显示切换按钮。后续如果增加不足四行的短介绍，可评估是否通过节点尺寸测量，仅在文本实际溢出时显示按钮。

### 6.3 评估交互细节

根据真机体验决定是否增加：

- 展开、收起状态图标。
- 更明确的按钮点击反馈。
- 收起后自动回到“详细介绍”标题附近。
- 在不影响长文本完整性的前提下加入过渡效果。

### 6.4 延续详情页信息优化

继续关注地址、地图、推荐理由和实用提醒的阅读顺序，结合后续用户测试评估是否需要进一步优化信息层级，但不改变现有数据结构和核心导航能力。
