# 《西安游迹》产品文档

> 项目名称：西安游迹（XI'AN TRAVEL GUIDE）
> 当前版本：v1.0.0（区域交互与实时搜索已完成）
> 更新日期：2026-07-23
> 文档状态：与当前已实现功能同步

## 1. 项目定位

《西安游迹》是一款以西安示意地图为入口，按区域查看景点、美食，并辅助个人旅行规划的轻量微信小程序。

项目主要服务于西安自助旅行场景。用户可以从首页地图或区域卡片进入区域攻略，通过首页与区域页的实时搜索快速查找景点和美食，并通过全屏地图、周边探索和微信原生地图能力辅助现场游览。

项目保持轻量、清晰和易维护：

- 使用微信原生小程序开发。
- 使用微信原生 `map` 组件。
- 使用本地 JavaScript 文件驱动页面内容。
- 不使用服务器、数据库和用户登录系统。
- 不接入第三方地图 SDK，不自行实现路线规划。
- 地图位置仅供旅行参考，实际入口和门店信息以官方或商家最新信息为准。

## 2. 目标用户与使用场景

### 2.1 目标用户

- 前往西安自助旅行的用户。
- 希望快速了解西安景点与当地美食的同行朋友。
- 需要一份轻量旅行资料和位置参考的个人用户。

### 2.2 当前用户场景

#### 自助旅行查询

- 从首页了解当前收录的西安旅行区域。
- 按区域查看景点与美食，减少分散查询。
- 在出发前快速浏览区域特色和旅行内容。

#### 快速查看景点信息

- 查看景点摘要和详细介绍。
- 查看开放时间、门票信息和建议游览时长。
- 查看地址、推荐理由、标签和注意事项。

#### 查找当地美食

- 按区域查看当前收录的西安美食。
- 查看美食介绍、参考价格、推荐店铺和地址。
- 查看推荐理由、标签和注意事项。

#### 实时搜索景点与美食

- 在首页地图下方直接展开搜索输入框，不跳转页面。
- 在区域页搜索时只匹配当前区域的景点和美食。
- 输入过程中按名称实时联想，名称开头匹配优先，再展示其他包含匹配。
- 联想面板最多展示 5 条结果，景点进入景点详情，美食进入美食详情。

#### 旅行过程中辅助定位

- 用户主动点击首页定位按钮后，在地图上显示当前位置。
- 进入横屏全屏地图浏览精选景点和地图特殊地点。
- 从景点详情查看当前景点及 3 公里内的周边景点。
- 对具有可靠参考坐标的美食查看门店参考位置。
- 从景点或美食详情调用 `wx.openLocation` 打开微信地图。

#### 出发前核对官方预约

- 在每个景点详情页查看预约状态、官方渠道、操作说明、官方来源和核实日期。
- 对官方微信公众号复制准确名称，再在微信中搜索进入。
- 对官方网页复制网址后自行访问，不绕过小程序 `web-view` 业务域名限制。
- 对开放式街区明确显示“无需预约”。
- 对无法可靠确认的景点明确显示“暂未确认”，不提供猜测入口。

## 3. 当前产品结构

```text
首页
├── 原生地图 → 精选景点 / 特殊地点 / 横屏全屏地图 / 用户定位
├── 内嵌实时搜索 → 景点详情 / 美食详情
└── 点击区域卡片 → 区域攻略
    ├── 当前区域内嵌实时搜索 → 景点详情 / 美食详情
    ├── 景点分类 → 互斥展开景点列表 → 景点详情
    └── 美食分类 → 互斥展开美食列表 → 美食详情

景点详情 → 官方预约信息 / 周边探索地图 / 打开微信地图
美食详情 → 美食参考地图 / 打开微信地图
```

当前主要页面：

```text
pages/home          首页地图、内嵌实时搜索、区域入口和用户定位
pages/full-map      横屏全屏地图
pages/region        区域内搜索及景点、美食折叠列表
pages/search        保留的独立实时搜索页，当前无首页或区域入口
pages/detail        景点详情
pages/place-map     当前景点与附近景点地图
pages/food          美食分类展示
pages/food-detail   美食详情
pages/food-map      美食参考位置地图
```

## 4. 当前功能范围

### 4.1 已完成功能

#### 首页地图

- 使用微信原生地图展示西安区域。
- 默认显示西安市中心范围。
- 保留区域卡片作为主要区域入口。
- 提供放大按钮进入横屏全屏地图。

#### 精选地点 marker

- 首页与全屏地图共同展示西安钟楼、大雁塔·大慈恩寺、陕西历史博物馆、洒金桥和大唐不夜城 5 个精选景点 marker。
- 精选配置统一保存在 `map-config.js`，两个页面复用同一数据源。
- 点击 marker 可以进入对应景点详情。
- marker 与景点 ID 使用明确映射，避免数据排序变化导致跳转错误。

#### 特殊地点系统

- `map-points.js` 管理只需要在地图展示、不属于景点内容体系的地点。
- 当前特殊地点为西安奥体中心，显示在首页和全屏地图。
- 特殊地点 marker 点击后不进入详情，也不加入区域攻略和周边探索。

#### 横屏全屏地图

- 页面采用横屏配置，地图占满可用屏幕。
- 复用首页精选景点和特殊地点数据。
- 保留精选景点详情跳转与用户主动定位能力。

#### 用户当前位置

- 用户点击首页定位按钮后获取 GCJ-02 位置。
- 定位成功后地图移动到用户当前位置并显示位置点。
- 页面加载时不自动申请定位权限。
- 不保存用户位置。

#### 区域攻略

- 当前包含 5 个旅行区域。
- 根据区域 ID 筛选当前区域的景点和美食。
- 景点和美食分类默认全部收起，点击后互斥展开，再次点击当前分类时收起。
- 区域页内嵌搜索只使用当前区域筛选后的景点和美食数组。
- 景点、美食分别进入对应详情页。
- 内容为空时显示友好状态。

#### 实时搜索系统

- 首页与区域页均在当前页面原地展开输入状态，不跳转 `pages/search`。
- `search-utils.js` 统一搜索景点和美食的 `name` 字段。
- 名称开头匹配优先，再展示其他包含匹配，结果最多 5 条。
- 搜索结果使用统一结构保存类型、图标、名称和详情地址。
- 首页搜索覆盖全部 `places.js` 与 `foods.js`；区域搜索只使用当前 `regionId` 已筛选的数据。
- 空关键词隐藏联想面板，无结果时显示明确提示。
- `pages/search` 继续保留并注册，复用同一公共工具，但当前首页和区域页均无入口跳转至该页。

#### 景点详情

- 展示名称、所属区域、摘要和详细介绍。
- 展示开放时间、门票信息和建议游览时长。
- 展示地址、推荐理由、标签和注意事项。
- 有有效坐标时显示“查看参考位置”和“打开地图导航”。
- 展示预约状态、官方渠道、操作说明、官方来源和核实日期。
- 官方公众号支持弹窗显示并复制名称；官方网页支持复制网址。
- 无需预约和暂未确认的景点不显示操作按钮。

#### 景点官方预约

- 13 个景点均具有统一 `reservation` 数据。
- 当前分布为 4 个官方微信公众号、2 个官方网页、4 个无需预约、3 个暂未确认。
- 所有来源在 `2026-07-22` 通过景区、博物馆或政府官方页面核实。
- 没有可靠确认的 `appId` 和 `path`，因此当前不启用 `wx.navigateToMiniProgram()`。
- 官方网页不通过 `web-view` 直接打开，只提供网址复制和限制说明。
- 复制、弹窗失败时显示明确提示；用户取消时不执行复制。

#### 美食详情

- 展示名称、类型、所属区域和摘要。
- 展示参考价格、详细介绍、推荐店铺和地址。
- 展示推荐理由、标签和注意事项。
- 9 条美食均使用本地 JPG 封面图，区域页和美食列表复用同一封面路径。
- 详情页通过轮播展示图片组，保留指示点，并支持点击进入微信全屏图片预览。
- 有有效坐标时显示“查看参考位置”和“打开地图导航”。

#### 景点周边探索地图

- 保留当前景点 marker，并从 `places.js` 中查找 3 公里内最多 5 个附近景点。
- 排除当前景点，按两点直线距离从近到远排序。
- 点击当前景点 marker 不跳转，点击附近景点 marker 进入对应详情。
- 使用 `include-points` 自动调整地图范围。
- 坐标或景点无效时显示错误状态。
- 明确说明位置只供旅行参考。

#### 美食参考位置地图

- 只对具有有效坐标的美食开放入口。
- 显示单个美食 marker、推荐店铺和地址。
- 点击 marker 时校验页面来源，并返回上一层对应的美食详情。
- 坐标或美食无效时显示错误状态。
- 明确说明实际门店信息以商家最新情况为准。

#### 地图导航

- 景点和具有有效坐标的美食详情页显示导航按钮。
- 统一通过 `location-utils.js` 调用微信原生 `wx.openLocation`。
- 景点使用景点名称，美食优先使用推荐店铺名，否则使用美食名称。
- 无有效经纬度时隐藏导航入口。
- 当前特殊地点仅用于地图展示，不提供详情页导航入口。

### 4.2 暂未完成功能

## 5. 当前内容与数据原则

截至 v1.0.0，项目收录：

- 5 个旅行区域。
- 13 个景点。
- 9 条美食。
- 13 个景点均具有参考坐标。
- 2 条美食具有已确认的参考坐标。

当前区域包括：

- 雁塔区。
- 碑林区。
- 莲湖区。
- 钟鼓楼区域。
- 临潼区。

“钟鼓楼区域”是为了旅行浏览建立的游览区域，不代表正式行政区划。

数据管理原则：

- 区域数据保存在 `miniprogram/data/regions.js`。
- 景点数据保存在 `miniprogram/data/places.js`。
- 美食数据保存在 `miniprogram/data/foods.js`。
- 地图默认中心、缩放级别、精选地点和周边参数保存在 `miniprogram/data/map-config.js`。
- 地图特殊地点保存在 `miniprogram/data/map-points.js`，不并入景点或区域数据。
- 坐标校验、距离计算与附近景点筛选由 `miniprogram/utils/map-utils.js` 提供。
- 打开微信地图的能力由 `miniprogram/utils/location-utils.js` 统一提供。
- 景点与美食名称联想由 `miniprogram/utils/search-utils.js` 统一提供。
- 页面通过稳定英文 `id` 和 `regionId` 查询、关联数据。
- 景点预约信息通过 `reservation` 对象统一管理，不写入美食数据。
- 每条预约信息保存渠道状态、渠道类型、平台名称、操作参数、说明、官方来源、来源网址和核实日期。
- 未确认的小程序 `appId`、页面 `path`、网址或公众号名称保持为空，不进行推测。
- 景点和美食使用独立的数据结构及详情页面。
- 不在多个页面重复手写同一份旅行内容。
- 景点开放时间、门票和美食门店情况可能变化，需要在出发前再次核对。
- 未确认的门店坐标不强行补充。

## 6. 当前明确不做

v1.0.0 不包含：

- 自行实现路线规划或接入第三方地图 SDK。
- 基于用户当前位置的动态附近搜索；当前仅实现以景点为中心的固定半径周边探索。
- 后台持续定位。
- 用户位置保存。
- 酒店模块。
- 票务、酒店预订或支付。
- 服务器、数据库和后台管理。
- 用户注册、登录、收藏、评论和评分。
- 自动获取实时天气、客流或营业状态。
- 绕过微信小程序业务域名限制直接打开未配置的预约网页。
- 使用携程、美团、飞猪等第三方平台作为景点官方预约入口。

## 7. 后续方向

### 7.1 三日行程

- 根据真实旅行计划确定三天安排。
- 行程节点优先关联已有景点或美食 ID。
- 避免在行程数据中重复保存详情内容。

### 7.2 图片完善

- 为景点选择有可靠来源及使用权限的图片。
- 在接入前压缩图片，控制小程序包体积。
- 保持首页、卡片和详情页视觉风格统一。

### 7.3 用户体验优化

- 在微信开发者工具和真机中持续检查地图、横屏、定位、导航、实时搜索和页面跳转。
- 根据朋友体验反馈优化信息层级与交互提示。
- 临近旅行日期时核对开放时间、门票、预约和门店信息。
- 在微信开发者工具和真机测试预约弹窗、剪贴板、长文本和快速重复点击。

## 8. v1.0.0 完成标准

用户应当能够：

1. 打开小程序并看到西安地图和区域入口。
2. 在首页或横屏全屏地图浏览精选景点和特殊地点。
3. 点击精选景点 marker 查看对应景点详情。
4. 主动获取并在地图中显示当前位置。
5. 在首页实时搜索全部已收录景点和美食。
6. 在区域页搜索当前区域内容，并按需展开景点或美食列表。
7. 查看景点信息、当前位置与附近景点，并打开微信地图。
8. 查看景点的官方预约状态，并复制已确认的公众号名称或官方网址。
9. 查看美食信息、可用的参考位置，并打开微信地图。
10. 无需注册或登录即可完成上述操作。

三日行程和景点正式图片不属于当前 v1.0.0 完成标准。
