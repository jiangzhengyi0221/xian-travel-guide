# 《西安游迹》产品文档

## 1. 项目介绍

### 1.1 项目名称

《西安游迹》

### 1.2 项目定位

《西安游迹》是一款个人旅游攻略微信小程序，用于整理西安旅行中的区域、景点、美食、酒店和三日行程信息。

小程序主要供开发者本人在西安旅行时查看，也可以分享给同行朋友。它不是实时地图或公共旅游平台，而是一份结构清晰、方便随时打开的个人旅行手册。

### 1.3 项目目标

- 通过一张西安市示意地图快速进入不同区域的攻略页面。
- 集中查看各区域的景点、美食和酒店信息。
- 查看地点的图片、简介、地址和推荐理由。
- 按天查看已经安排好的三日旅行行程。
- 使用尽量简单的技术完成一个可以真实使用和分享的小程序。

### 1.4 技术原则

第一版遵循以下原则：

- 使用微信原生小程序开发。
- 使用本地 JSON 文件保存数据。
- 使用静态示意地图图片，不接入地图 API。
- 不使用服务器。
- 不使用数据库。
- 不使用用户登录系统。
- 不开发后台管理功能。
- 优先保证内容清楚、操作简单和项目容易维护。

## 2. 用户需求

### 2.1 目标用户

- 即将前往西安旅行的开发者本人。
- 同行或收到小程序分享的朋友。

### 2.2 使用场景

#### 出发前

- 了解准备游览的区域。
- 查看计划中的景点、美食和酒店。
- 检查三日行程安排是否完整。

#### 旅行中

- 快速查看某个地点的简介和地址。
- 查看该地点被推荐的原因。
- 按照三日行程确认下一站。

#### 分享时

- 将小程序分享给朋友。
- 让朋友查看相同的区域攻略和行程安排。

### 2.3 核心用户需求

1. 用户希望从直观的西安示意地图进入不同区域。
2. 用户希望快速了解一个区域内有哪些景点、美食和酒店。
3. 用户希望点击地点后查看更完整的信息。
4. 用户希望集中查看三天的旅行安排。
5. 用户希望页面简单，不需要注册或登录即可使用。

## 3. 页面结构

第一版包含四个页面。

### 3.1 首页

首页是小程序的主要入口，展示：

- 小程序名称和简短介绍。
- 西安市示意地图图片。
- 雁塔区、碑林区、莲湖区等可点击区域。
- 地图下方的区域入口卡片。
- “我的三日行程”入口。

点击地图上的区域或区域卡片后，进入对应的区域攻略页。

地图上的点击区域只作为页面入口，不要求精确还原行政区边界。为避免用户不知道地图可以点击，地图下方同时提供清晰的区域卡片。

### 3.2 区域攻略页

区域攻略页展示当前区域的旅行内容，包括：

- 区域名称。
- 区域封面图片。
- 区域简短介绍。
- 景点、美食、酒店三个分类入口。
- 当前分类下的地点列表。

所有区域共用同一个页面，根据用户选择的区域显示不同内容，不为每个区域重复制作一套页面。

### 3.3 地点详情页

地点详情页由景点、美食和酒店共同使用，展示：

- 地点图片。
- 地点名称。
- 地点类型。
- 所属区域。
- 地点简介。
- 详细地址。
- 推荐理由。
- 可选的实用备注。

实用备注可以根据地点类型记录不同内容，例如景点的建议游览时长、美食的推荐菜品或酒店的入住理由。

### 3.4 我的三日行程页

三日行程页按照第一天、第二天、第三天展示旅行计划。每个行程节点可以包含：

- 时间或时间段。
- 活动名称。
- 关联地点。
- 简短提醒。

如果行程节点关联了已有地点，用户可以点击进入该地点的详情页。

### 3.5 页面关系

- 首页进入区域攻略页。
- 区域攻略页进入地点详情页。
- 首页进入三日行程页。
- 三日行程页可以进入地点详情页。
- 用户可以使用小程序自带的返回功能回到上一页。

## 4. 功能列表

### 4.1 首页功能

- 展示西安市示意地图。
- 展示可浏览的区域名称。
- 点击地图区域进入区域攻略页。
- 点击区域卡片进入区域攻略页。
- 进入我的三日行程页。

### 4.2 区域攻略功能

- 展示区域基本信息。
- 在景点、美食和酒店之间切换。
- 展示当前分类的地点列表。
- 点击地点卡片进入详情页。
- 分类没有内容时显示友好的空状态提示。

### 4.3 地点详情功能

- 根据地点编号展示对应内容。
- 展示图片、名称、简介、地址和推荐理由。
- 展示与地点类型有关的实用备注。
- 支持返回区域页或行程页。

### 4.4 三日行程功能

- 按天展示行程。
- 按时间顺序展示当天安排。
- 展示活动提醒或注意事项。
- 点击已关联的地点进入详情页。

### 4.5 分享功能

- 使用微信小程序自身的页面分享能力，将首页或内容页分享给朋友。
- 第一版只考虑微信内分享，不生成独立网页。

## 5. 数据结构

第一版数据保存在本地 JSON 文件中，建议分为区域、地点和行程三类。以下内容只说明需要保存的信息，不表示具体代码格式。

### 5.1 区域数据

| 字段 | 用途 | 是否必需 |
| --- | --- | --- |
| 区域编号 | 唯一识别一个区域 | 是 |
| 区域名称 | 例如雁塔区、碑林区 | 是 |
| 区域简介 | 简要说明区域特色 | 是 |
| 封面图片 | 区域页使用的图片 | 是 |
| 推荐标签 | 例如历史、夜景、美食 | 否 |
| 显示顺序 | 控制首页区域排列顺序 | 否 |

### 5.2 地点数据

| 字段 | 用途 | 是否必需 |
| --- | --- | --- |
| 地点编号 | 唯一识别一个地点 | 是 |
| 所属区域编号 | 确定地点属于哪个区域 | 是 |
| 地点类型 | 景点、美食或酒店 | 是 |
| 地点名称 | 地点的正式名称 | 是 |
| 简介 | 简要介绍地点 | 是 |
| 地址 | 供旅行时查看 | 是 |
| 推荐理由 | 说明为什么值得前往 | 是 |
| 主图 | 列表和详情页的主要图片 | 是 |
| 更多图片 | 详情页可选图集 | 否 |
| 推荐标签 | 例如必去、拍照、早餐 | 否 |
| 实用备注 | 时间、预算、预约等提醒 | 否 |

不同地点类型可以在实用备注中记录不同信息：

- 景点：开放时间、门票、建议游览时长、是否需要预约。
- 美食：推荐菜品、营业时间、参考预算、是否需要排队。
- 酒店：价格范围、入住理由、交通和附近设施。

### 5.3 行程数据

| 字段 | 用途 | 是否必需 |
| --- | --- | --- |
| 天数编号 | 表示第一天、第二天或第三天 | 是 |
| 日期 | 对应的旅行日期 | 否 |
| 当天主题 | 概括当天安排 | 是 |
| 当天说明 | 当天整体提醒 | 否 |
| 行程节点 | 保存当天按顺序排列的活动 | 是 |

每个行程节点包含：

| 字段 | 用途 | 是否必需 |
| --- | --- | --- |
| 时间 | 例如上午或 09:00 | 是 |
| 活动名称 | 当前安排的名称 | 是 |
| 地点编号 | 关联已有地点详情 | 否 |
| 备注 | 预约、交通或用餐提醒 | 否 |
| 显示顺序 | 控制当天活动排列顺序 | 是 |

行程应尽量引用已有地点编号，避免在行程数据中重复保存地点简介和地址。

### 5.4 图片资源

- 第一版优先使用项目内的本地图片，减少网络依赖。
- 每个地点先准备一张经过压缩的主图，后续再决定是否增加图集。
- 图片文件名应简短、统一，并能看出对应地点。
- 应确认图片来源和使用权限。
- 添加图片时要注意控制小程序包体积。

## 6. 开发计划

### 阶段一：确定内容清单

工作内容：

- 确定第一版区域。
- 为每个区域列出景点、美食和酒店。
- 写好地点简介、地址和推荐理由。
- 排好三日行程。
- 收集并压缩需要使用的图片。

完成标准：所有计划展示的内容都已经列入清单，主要文字和图片不再缺失。

### 阶段二：绘制页面草图

工作内容：

- 绘制首页草图。
- 绘制区域攻略页草图。
- 绘制地点详情页草图。
- 绘制三日行程页草图。
- 确认每个按钮点击后进入哪个页面。

完成标准：可以通过页面草图完整讲清楚用户从首页到详情页的使用过程。

### 阶段三：准备示意地图和视觉素材

工作内容：

- 准备适合手机竖屏显示的西安市示意地图。
- 在地图上清晰标出第一版区域。
- 确定地图点击区域的大致位置。
- 统一封面图、地点图和图标的风格。

完成标准：地图在手机尺寸下仍然清晰，区域名称容易识别和点击。

### 阶段四：创建微信原生小程序项目

工作内容：

- 准备微信小程序 AppID。
- 使用微信开发者工具创建项目。
- 创建首页、区域页、详情页和行程页。
- 配置基本页面导航。

完成标准：四个空页面都可以正常打开和返回。

### 阶段五：完成静态页面

工作内容：

- 完成四个页面的基础布局。
- 完成区域卡片、地点卡片和行程节点的显示样式。
- 使用少量测试内容验证页面效果。

完成标准：不依赖完整数据，也可以走通首页、区域页、详情页和行程页。

### 阶段六：接入本地数据

工作内容：

- 整理区域 JSON 数据。
- 整理地点 JSON 数据。
- 整理三日行程 JSON 数据。
- 根据区域和分类显示地点。
- 根据地点编号显示详情。
- 将行程节点与地点详情关联。

完成标准：修改本地数据后，页面内容可以正确更新，不需要在多个页面重复修改同一地点的信息。

### 阶段七：真机测试和内容检查

工作内容：

- 在真实手机上检查地图和图片显示效果。
- 检查地图区域是否容易点击。
- 检查分类、页面跳转和返回功能。
- 检查所有地点是否打开正确的详情。
- 检查地址、开放时间和预约信息。
- 检查图片是否清晰且不会导致项目体积过大。

完成标准：本人和至少一位朋友可以顺利完成主要操作，没有明显内容错误。

### 阶段八：体验版和发布准备

工作内容：

- 上传体验版并邀请朋友测试。
- 根据反馈修正文字、图片和操作问题。
- 准备小程序名称、图标、简介和必要的发布资料。
- 出发前再次核对可能发生变化的旅游信息。

完成标准：小程序可以在旅行中稳定使用并分享给朋友。

## 7. 第一版范围

### 7.1 第一版必须完成

- 一个西安市示意地图首页。
- 雁塔区、碑林区、莲湖区三个区域入口。
- 一个共用的区域攻略页。
- 景点、美食和酒店三个分类。
- 一个共用的地点详情页。
- 图片、名称、简介、地址和推荐理由展示。
- 一个三日行程页。
- 行程中的地点可以进入已有地点详情。
- 本地 JSON 数据读取。
- 基本的微信页面分享。

### 7.2 建议内容数量

为控制初学者的工作量，建议第一版准备：

- 3 个区域。
- 每个区域 3 个景点。
- 每个区域 3 个美食。
- 每个区域 1 至 2 个酒店。
- 1 份三日行程。
- 总地点数量控制在 20 至 25 个左右。

### 7.3 第一版明确不做

- 地图 API 和实时地图。
- GPS 定位和实时导航。
- 服务器和云函数。
- 数据库和后台管理。
- 用户注册和登录。
- 收藏、评论和评分。
- 在线搜索和复杂筛选。
- 用户自行编辑行程。
- 票务、酒店预订或支付。
- 自动获取实时天气、客流或营业状态。
- 精确的不规则行政区边界点击。

### 7.4 第一版完成标准

第一版完成后，用户应当能够：

1. 打开小程序并看到西安示意地图。
2. 从地图或区域卡片进入指定区域。
3. 查看该区域的景点、美食和酒店。
4. 打开任意地点并查看完整介绍。
5. 查看三日行程并从行程进入地点详情。
6. 无需登录即可完成全部操作。
7. 将小程序页面分享给朋友使用。

如果以上流程能够在真机上稳定完成，第一版就达到目标。其他功能应等旅行实际使用后，再根据真实需求决定是否增加。
